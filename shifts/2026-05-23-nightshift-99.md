- **All queries go through Cascades** (line 311 of connection.go). The plan visitor's pushdown chain (`select_query_full.go`) is dead code for `database/sql` driver queries.
- **EXPLAIN uses naive path**, not Cascades. EXPLAIN output doesn't reflect the Cascades physical plan.
- **PlanningCostModelLess criterion #7 has swapped return values** â€” `comparePrimaryScanVsIndexScan` returns 1 (index wins) when it should return -1 (primary wins), and vice versa. Hidden by criterion #2's `.Total()` vs `.Cardinality` asymmetry. Not fixed this shift to avoid cascading regressions.
