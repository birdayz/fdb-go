## What was done

### SchemaTemplateBuilder (`pkg/relational/core/metadata/builder.go`) — NEW
- `Builder` constructs `RecordLayerSchemaTemplate` from SQL-level table/column/PK definitions without a pre-compiled `.proto` file
- Builds `FileDescriptorProto` dynamically from `api.DataType` column specs (no union descriptor — sidesteps global proto registry for dynamically-created types)
- Column labels: NOT NULL → `LABEL_REQUIRED`, NULL → `LABEL_OPTIONAL` (proto2 semantics)
- Primary key: `RecordTypeKey()` prefix when !intermingle, then `Field(colName)` per PK column
- All 7 data types mapped: BOOLEAN/INTEGER/LONG/FLOAT/DOUBLE/STRING/BYTES
- 8 unit tests covering: basic, multi-table, all column types, empty PK error, empty name error, empty tables error, nullable vs NOT NULL round-trip

### CREATE/DROP SCHEMA TEMPLATE SQL wired
- `execCreateSchemaTemplate` + `execDropSchemaTemplate` in `EmbeddedConnection`
- `parseTableDefinition` reads columns + PKs from ANTLR parse tree; infers nullable from `NullColumnConstraintContext.NullNotnull().NOT()`
- Grammar syntax: `CREATE SCHEMA TEMPLATE name CREATE TABLE T (col TYPE [NOT NULL], ..., PRIMARY KEY (cols))`
- 2 FDB integration tests: create+drop, duplicate template fails

### Review feedback addressed (commit 1dfdcae)
- `sync.Once` → `sync.Mutex + bool` in `ensureCatalogInit` — transient init failures retried
- Extracted `ensureCatalogInit` helper (dedup from Ping + runDDL)

- `parseSchemaIdentifier`: single-segment absolute paths (`/schema`) now return error
- `validateDatabasePath`: bare `/` and trailing-slash paths now rejected
- Tests added for all new invalid cases

### Full DDL flow integration test
