### 2. Connection monitor (new feature, C++ conformant)

Port of C++ FlowTransport.actor.cpp connectionMonitor() (lines 641-721):
- Outer loop: 750ms check, idle skip, 750ms jittered delay, send PING
- Inner retry loop: 2s timeout per round, retry if ANY bytes arrived, kill only on true silence
- PING reply registered via PrepareReply — matches C++ `pingRequest.reply.getFuture()`
- Idle connections (no pending requests) untouched
- Zero benchmark overhead

Key insight: C++ uses `timeouts` counter only for logging, NOT for kill decisions. The kill condition is solely `bytesReceived unchanged`. Our inner retry loop matches this exactly.
