- **`e7eed5d` Parse() wrapper + caseInsensitiveCharStream**: `parser.Parse(sql) (IRootContext, error)` with a collecting `antlr.ErrorListener` that turns every syntax error into one `line:col: msg` line of an `*api.Error` with `ErrCodeSyntaxError`. `caseInsensitiveCharStream` embeds `antlr.CharStream` and overrides only `LA()` to upper-case; EOF (-1) and 0 pass through. Source text preserved for `GetText()` calls. **Renamed the generated package from `parser` to `antlrgen`** so the wrapper (also `package parser`) doesn't shadow its own import. 9 test functions: valid CRUD / DDL / transaction / empty-input, mixed-case, single syntax errors, line:col format, stray-char rejection, multi-error ordering, case-folding math, EOF passthrough.

Schema templates get wrapped in `CREATE SCHEMA TEMPLATE test_template {...}`. Skip filters: `!! … !!` macros, `SHOULD ERROR`-style sentinels, and entries whose sibling declares `error:`. **1587 / 1587 filtered statements parse cleanly.** Gated by `//go:build yamsql` so Bazel's sandbox doesn't need to see the submodule; `just smoke-yamsql` runs it via plain `go test`.

### Phase 2 — catalog bridge (4 commits)

- **`6ce92de` RecordLayerSchemaTemplate**: first concrete `api.SchemaTemplate` backed by `*recordlayer.RecordMetaData`. Mirrors Java's `RecordLayerSchemaTemplate.fromRecordMetadata` path.
  - Types in `pkg/relational/core/metadata/`: `RecordLayerSchemaTemplate`, `RecordLayerTable`, `RecordLayerColumn`, `RecordLayerIndex`, `recordLayerSchema`.
  - `proto_types.go` translates protobuf field descriptors to `api.DataType`: scalar → primitive, enum → EnumType with declared values, message → recursive StructType, repeated → ArrayType, map → UnresolvedType. All proto fields nullable by default (matches Java).
  - Eager construction: tables + flat index-name list materialised up-front. `FindTable` is O(1).
  - **Java-compliance divergences caught during self-audit**: (1) stopped duplicating universal indexes into each per-table `Indexes()` (Java's `RecordMetadataDeserializer.generateTableBuilder` only attaches per-type indexes); (2) `Accept()` overrides the package-level `VisitSchemaTemplateTree` to cascade through tables → indexes → columns → routines → views → finish, matching Java's `accept()` body exactly.
  - 11 test functions covering construction, sorted ordering, type translation, universal-index non-duplication assertion, visitor cascade counts, schema generation.
- **`a4b0387` doc on deferred items**: `pkg/relational/core/metadata/doc.go` lists 3 unimplemented Java behaviours (`NullableArrayTypeUtils.describesWrappedArray`, `primaryKeyHasRecordTypePrefix → intermingleTables`, sparse-index `IsSparse`) and 1 closed follow-up (UUID detection).
- **`526d7ae` UUID short-circuit**: `isUUIDDescriptor(md)` detects the canonical `com.apple.foundationdb.record.UUID` message descriptor and surfaces it as `UUIDType` instead of a two-field struct. Matches Java's `fromProtoType()` short-circuit. 2 tests.
- **`09072a4` Schema delegation methods**: `api.Schema` grew four methods to match Java's Schema.java default methods: `Tables() ([]Table, error)`, `Views() ([]View, error)`, `Indexes() (map[string][]string, error)`, `InvokedRoutines() ([]InvokedRoutine, error)`. Bridge's `recordLayerSchema` delegates each to its template. **Shape divergence note**: `Schema.Indexes()` returns the `(table → index names)` map (matches Java's Multimap), while `SchemaTemplate.Indexes()` returns the flat `[]string` — same name, different shape, mirrors Java exactly. Pinned with one test.
- **`90ae2c2` wrapped-array unwrap**: `NullableArrayTypeUtils.describesWrappedArray` Java equivalent — `message M { repeated R values = 1; }` wrapper messages now unwrap to `ArrayType(element)`. Closes another deferred item. 4 tests built synthetic descriptors via `protodesc.NewFile`.
- **`c4b59af` IntermingleTables()**: added `api.SchemaTemplate.IntermingleTables()` → `!RecordMetaData.PrimaryKeyHasRecordTypePrefix()`.
- **`e686af1` version decoupling**: Java takes the template version externally in `fromRecordMetadata(md, name, version)`. Added `NewRecordLayerSchemaTemplateWithVersion` alongside the default constructor; `Version()` now returns the pinned value instead of `RecordMetaData.Version()`. Catalog can drive schema versioning independently of the storage metadata. 1 test.
- **`081abd6` map-field guard in `unwrapWrappedArray`**: reviewer-caught defensive gap. A `map<string,int32> values = 1;` field would otherwise pass the wrapper shape checks because protobuf maps are lowered to repeated synthetic messages.
- **`827ae5d` table StructDataType nullable=true**: Java's `RecordLayerTable.getDatatype()` emits `nullable=true`. Ours was emitting `false`. Track Java's emitted bit, not its acknowledged TODO.
- **`79810b1` proto2 REQUIRED label nullability**: Java's `Type.fromProtoType` respects proto field labels (`REQUIRED` → non-nullable, `OPTIONAL`/`REPEATED` → nullable). Ours hard-coded `nullable=true`. Rare in real schemas but the divergence is fixed now.
- **`616fd66` IsSparse follows `predicate != null`**: Java's rule exactly. Was hard-coded `false` with a comment blaming recordlayer, but recordlayer.Index.Predicate was already there.

## Java-compliance audit status

Per the user's "100% Java compliant" directive, here's what's matching and what's still deferred.

**Matches Java exactly:**
- Parser grammar: byte-verbatim except for one documented deletion (Java `notifyListeners` action which the Go target can't compile).
- SQL-dialect recognition: 1587/1587 corpus statements parse.
- `RecordLayerSchemaTemplate.accept()` cascade: start → visit → tables → routines → views → finish.
- Per-type vs universal indexes: universal appears in `Indexes()` flat list only, NOT per-table. Pinned by a dedicated test.
- UUID detection: `com.apple.foundationdb.record.UUID` surfaces as `UUIDType`.
- Proto type → DataType mapping: integer variants (int32/sint32/sfixed32/uint32/fixed32 → Integer; 64-bit variants → Long), float/double, string/bytes, enum with values, message → StructType, repeated → ArrayType.
- Schema default-method delegation: Tables / Views / Indexes / InvokedRoutines all delegate to the template.
- `Schema.Indexes()` returns `map[string][]string` (Java Multimap shape), not the flat list.

**Known divergences (documented in `pkg/relational/core/metadata/doc.go`):**

None known on the primary path after dayshift-25's audit pass. Closed:
- UUID descriptor short-circuit (`com.apple.foundationdb.record.UUID` → `UUIDType`)
- `NullableArrayTypeUtils.describesWrappedArray` — wrapper messages unwrap to ArrayType
- Map-field defensive guard inside `unwrapWrappedArray`
- `primaryKeyHasRecordTypePrefix → intermingleTables` template flag
- `Schema.Tables/Views/Indexes/InvokedRoutines` delegated methods (Java default-method equivalents)
- `fromRecordMetadata(md, name, version)` version-decoupling contract
- Table's `StructDataType` emits `nullable=true` (Java tracked, including its acknowledged TODO)
- Proto2 `REQUIRED`-label nullability (was hard-coded `true` for all labels)
- `IsSparse()` now follows Java's `predicate != null` rule (using recordlayer.Index.Predicate)

Edge cases the Java side has acknowledged TODOs about (table-level nullable=true, maps as UnresolvedType, auxiliary types from Union enums) mirror Java's emitted behaviour rather than Java's intent — the user wants 1:1 compliance with what Java actually does today.

## Notes for next shift

- **`just gazelle` is part of the pre-commit hook**, but it's easy to miss gazelle updates if a test file is added via `Write` without triggering the hook's `just gazelle` step before `git commit`. The 526d7ae commit's CI caught this. The hook generally catches it; stay vigilant on test-file adds.
- **ANTLR runtime has no 4.13.2 on pkg.go.dev** — if you ever bump the ANTLR tool, keep it at 4.13.1 until antlr4-go publishes a matching runtime tag.
- **`Schema.Indexes()` vs `SchemaTemplate.Indexes()` return shape divergence** is a landmine. Same method name, different shapes. Flagged in the interface doc comment; keep an eye when porting code that calls either.
- **The 3 deferred Java-compliance items** (NullableArrayTypeUtils / intermingleTables / IsSparse) are documented in `pkg/relational/core/metadata/doc.go`. Close them before declaring the bridge "complete" — a Java round-trip test will fail on (1) otherwise.
- **User wants `100% Java compliance`** (their exact words). Before committing any new api.* or core/metadata/ type, grep the corresponding Java source, compare method-by-method, and note any divergences with a justified `// Java-compliance: …` comment.
- **Memory saved on the relational port is in `project_relational_layer_port.md`.** Update it when the Phase 2 concrete impl lands if anything surprising came up.
