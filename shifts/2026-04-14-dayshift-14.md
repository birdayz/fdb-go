# Dayshift-14 Handover

**Date:** 2026-04-14, started ~07:10 CEST
**PR:** #52

## What was done

### 1. CI test cache invalidation fix (main task)

**Root cause:** `bench-ci` used `bazelisk test` with `--nocache_test_results` and different `--test_timeout`/`--test_arg` flags. Bazel stores one action cache entry per test target — the bench step's execution overwrote the test step's cached result. Every CI run re-executed `recordlayer_test` (~45s) and `types_test` (~0.1s).

**Fix:** All bench recipes (`bench`, `bench-one`, `bench-ci`, `bench-compare`) now use `bazelisk run` (build binary + execute directly). This bypasses the test cache entirely.

**Verification:** 3 consecutive CI runs confirmed all 16 test targets are `(cached)`. Test step: 50s → 4.7s.

### 2. CI cleanup

- Removed redundant `--test_timeout=900` override from `ci.yml` and `nightly-coverage.yml`. `.bazelrc` already sets proper per-category timeouts.
- Removed `2>/dev/null` from `bazelisk info bazel-bin` calls (surface errors instead of silently failing).
- Moved `bazelisk build` outside `tee` block in `bench-ci` (cleaner bench-raw.txt).
- Added `timeout` wrapper to `bench-compare` recipe.

### 3. Bench-report false positive reduction

- Raised regression threshold from 5% to 10% (added `-threshold` flag).
- Only flags timing regressions when allocs/bytes also changed. Timing-only deltas on shared CI = VM noise, not code regression.

### 4. store_version.go test coverage

16 new specs for `packVersion`/`unpackVersion` roundtrip and `LoadRecordVersion`:
- 3 roundtrip variants (local=0, non-zero, max)
- Tuple format verification (Versionstamp type)
- 3 error handling tests (nil, invalid bytes, wrong type)
- LoadRecordVersion: non-existent record, after commit, snapshot, local cache, distinct local versions

### 5. Verification

- **Binding stress:** 20/20 seeds × 1000 ops = 0 failures
- **Fuzz:** FuzzFastUnpack 35M execs in 30s, 0 crashes
- **CI:** All 16 test targets cached on consecutive runs
