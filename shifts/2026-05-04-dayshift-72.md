## What was done

### #65 — Cascades is now the ONLY query engine

- **Cascades default:** All SELECT queries route through cascadesGenerator. QueryEngine enum + SetQueryEngine removed. `engine=cascades` DSN option removed.
- **Plan-time planning:** Planner runs at Plan() time with MaxTasks(1000). Cascades plans no longer fall back to naive on execution errors — only on plan-time failures.
- **LogicalAggregate → GroupByExpression:** Parses SUM/COUNT/AVG/MIN/MAX text into AggregateSpec + operand Values.
- **LogicalDistinct operator:** New logical operator for standalone SELECT DISTINCT. TranslateToCascades maps it to LogicalDistinctExpression.
- **Explicit CTE → nil fallback:** CTEs route through naive with explicit nil return.
- **deriveColumnsFromPlan:** Walks wrapper plans (filter/sort/distinct/etc) to find underlying scan. Handles projection plans via FieldValue extraction.
- **18 Cascades FDB integration tests:** scan, filter (eq/ineq/neq/or/multi-pred/string), projection, distinct, distinct+filter, sort with index, aggregate (count/sum), group by+sum, join, multi-column projection, index scan.

### PhysicalizeSortRule deleted — Java RemoveSortRule alignment

- Sort is a constraint (ImplementSortRule in PLANNING phase), not a physical operator
- physicalSortWrapper removed (dead code)
- 3 Go-only plangen tests deleted, 3 updated
