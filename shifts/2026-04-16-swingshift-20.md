## What was done

### Performance optimization port from PR #65 (external team collaboration)

An external team using fdb-record-layer-go hit bottlenecks and developed performance optimizations on the `worktree-example-usage-based-billing` branch (PR #65). This shift ported their safe optimizations to main via PR #66, with careful review of each change.

#### FDB Client (pkg/fdbgo)

| Change | Files | Impact |
|---|---|---|
| **ReplyHandle pool** | transport/conn.go + 8 client callers | Eliminates closure alloc per RPC. ~11 allocs/op saved. |
| **FrameReader** | transport/framing.go + conn.go readLoop | Persistent header buffer. -8 allocs/op. |
| **nil error short-circuit** | client/readpath.go | `isFutureVersionOrProcessBehind(nil)` avoids errors.As. -2 allocs/op. |
| **sync.Map -> map+RWMutex** | recordlayer/store_state_cache.go | Compiler `string([]byte)` optimization for subspace key cache. -1 alloc. |
| **Remove sync.Once from future** | fdb/future.go | Single-goroutine future doesn't need Once. -24 bytes/op. |

#### Record Layer (pkg/recordlayer)

| Change | Files | Impact |
|---|---|---|
| **ScanRecordsByType prefix scan** | store.go | **100x speedup** for typed scans on multi-type stores |
| **Inline SUM mutation** | atomic_mutation_index_maintainer.go | Avoids atomicMutationEntry + interface dispatch |
| **StoreBuilder subspace caching** | store_builder.go | Caches *storeSubspaceKeys on builder |
| **SetBytes/ClearBytes** | 4 index maintainers | Interface boxing avoidance in RANK/VERSION/TWL/PERMUTED |
| **Lazy store state loading** | store.go, store_batch.go, index_state.go | `ensureStoreStateLoaded()` with sync.Once for Build() path |
| **SetAssumeAllIndexesReadable** | store_builder.go | Zero-overhead Build() when all indexes known READABLE |
| **Skip uniqueness checks in InsertBatch** | store.go, store_batch.go, index_maintainer.go | **2.7x throughput** by eliminating GetRange per UNIQUE entry |

### Items deliberately NOT ported (correctness concerns)

- **selfConflicts shallow copy** in OnError — removes deep copy, risks use-after-pool-return
- **"Capture tx fields before closures" removal** in readpath.go — would reintroduce data races (commit 0927b0f)
- **commitDummyTransaction changes** — removes backoff, removes Set(), behavioral change
- **Location cache wide invalidation** — diverges from C++ (narrow invalidation per shard)
- **GRV cache SYSTEM_IMMEDIATE guard removal** — defensive code, risky to remove
- **Read conflict buffer inlining** — code duplication for no perf benefit (already pooled)
