### 3. embedded: derived-table WHERE in catalog-aware builder — `1f88d605`

`buildWherePredicate` previously declined on `sq.derivedQuery != nil`.
Now `buildWherePredicateForDerived` extracts the inner query's
selectQuery via the existing `extractFromQueryTerm` parser, walks
`projCols` / `projAliases`, looks up each bare-column projection in
the inner table's catalog entry, and synthesises a
`semantic.StaticTable` whose columns inherit the inner-table types.
The synthesised `ScopeSource` lets the walker resolve `x.val` against
the inner BIGINT column without the derived rows having to
materialise.

`buildDerivedTableSource` declines on shapes the seed Type system
can't reason about — derived-of-derived, joins, SELECT *,
aggregates, computed projections, qualified-star — and the caller
falls back to the text builder. Phase 4.0 Type hierarchy port
unlocks computed projections; the decline gate documents why.

### 4. cascades/values: Phase 4.0 Type hierarchy seed — `d4f26925`, `6fa1edfe`, `6bec402b`, `2fba3975`, `0b2f8c35`, `27f5b1d9`

Mirrors the bare-minimum surface of Java's
`com.apple.foundationdb.record.query.plan.cascades.typing.Type`.
Lives in `cascades/values/type.go` (~1000 LOC + ~1000 LOC of tests)
— above RFC-025's 300-LOC threshold for a `typing/` split, but
deferred until the next port (RelationType, plan-serialisation
hooks, the conversion lattice) actually needs the move.

Contents:

- `TypeCode` enum — Java's well-known codes (NULL, BOOLEAN, INT,
  LONG, FLOAT, DOUBLE, STRING, BYTES, VERSION, ENUM, RECORD, ARRAY,
  RELATION, ANY, NONE, UUID) plus UNKNOWN. `IsPrimitive` /
  `IsNumeric` helpers.
- `Type` interface — Code + IsNullable + Equals (structural) + String.
- `PrimitiveType` impl + canonical singletons (`NotNullInt`,
  `NullableString`, `NullType`, `UnknownType`, etc.).
  `NewPrimitiveType` panics on structured codes.
- `RecordType` (Field + Ordinal-based) — anonymous-field-allowed,
  duplicate-named-field panic, defensive-copy constructor,
  LookupField + GetField.
- `ArrayType` — ElementType (nullable supported), nested ARRAY-of-
  RECORD, nil-element-as-not-yet-inferred.
- `EnumType` — Name + Number values, dup-name AND dup-number
  panic, both lookups, declared-order-aware Equals.
- `WithNullability(t, nullable)` — flips a Type's nullability;
  returns canonical singletons for primitives, fresh copies for
  structured types.
- `Typed` interface (RichType()) — forward-compat hook for
  things-with-a-type, parallel to existing Value.Type() ValueType.
- `FromValueType(vt, nullable)` / `ToValueType(t)` — legacy bridge.
  `TypeInt` ↔ LONG (BIGINT default); `TypeFloat` ↔ DOUBLE; the
  reverse collapses LONG/INT and DOUBLE/FLOAT.
- `TypeRepository` — name → Type registry. Per-query / per-statement
  instance, not goroutine-safe. `TypeRegistrationError` structured
  error.
- `ValueRichType(v)` free function — bridges every Value impl to the
  rich Type without forcing the Value interface itself to gain a
  RichType() method (avoids a breaking change).

### 5. cascades/matching: Phase 4.2 matcher catalogue — `bec23171`, `9b93ae77`, `a3bea697`, `82a29849`, `c869c9a6`, `8eab9221`, `9502a2ff`

Five new matchers + one interface, substantially closing the
RFC-025-aligned matcher gap from Java's
`matching/structure/` package:

- **`SomeElementsMatcher`** (Java's `MultiMatcher.SomeMatcher`) —
  matches when at least one element binds the downstream. Empty
  input fails (asymmetric vs AllElementsMatcher's vacuous-true).
- **`AtLeastElementsMatcher`** (Java's `MultiMatcher.AtLeastMatcher`)
  — matches when at least N elements bind. minMatches=0 always
  matches; minMatches>0 + fewer matches → nil. Negative panics.
- **`EmptyCollectionMatcher`** (Java's `CollectionMatcher.empty`) —
  matches an empty []any input. id-counter pattern for distinct
  identity.
- **`TypedMatcher[T, U]`** (Java's
  `TypedMatcherWithExtractAndDownstream<T>`) — generic: type-asserts
  to T, applies extractor T → U, runs downstream on U. nil-extract /
  nil-downstream panic at construction.
- **`NegationMatcher`** (Java's `NotMatcher`) — inverts a downstream:
  succeeds when downstream fails, fails when it matches. Naming
  disambiguated from the pre-existing `predicateMatcher[*NotPredicate]`
  (which matches when the input IS a NotPredicate).
- **`SatisfyingMatcher[T]`** (Java's `PrimitiveMatchers.satisfies`)
  — generic: type-asserts to T, runs caller-supplied predicate(T).
  Use case: rule pattern constraints that don't fit a dedicated
  matcher.
- **`CollectionMatcher` interface** — closed-set marker interface
  satisfied by the four list-matchers (AllElements / Some /
  AtLeast / List / Empty) for type-safe combinator composition.

### 6. Bug fixes (all by code-inspection audit)

- **`CAST(int64 AS STRING)` on negatives** (`1540acf0`) — used
  `uitoa(uint64(i))` which reinterprets the sign bit; `CAST(-5 AS
  STRING)` produced "18446744073709551611". Fixed with
  `strconv.FormatInt`. Regression test pins -5 / 0 / MIN_INT64 /
  MAX_INT64.
- **`intToDec(MinInt64)` plan-cache-key collision** (`cadbcd4c`) —
  hand-rolled formatter used `n = -n` which overflows for MinInt64
  and produced just `"-"`. Since intToDec drives ExplainValue
  (the plan-cache key seam), wrong rendering would silently hash-
  collide distinct queries. Same fix pattern; same regression
  pinning.
- **`buildSchemaTemplateFromDDL` nil-deref** (`e1920e8d`) —
  short-circuit `len(stmts.AllStatement())` after `stmts == nil`
  check would panic. Split the guards.
  `TestGoEngine_CatalogAware_StatementlessDDL` (whitespace-only,
  semicolon-only) is the regression sentinel.
- **map-path IN-list expression evaluation** (`c499cec1`) — TODO
  in eval_predicate_map.go silently dropped arithmetic / function-
  call list elements (only PredicatedExpression's atom was
  evaluated). Switched to `evalExprOnMap` to match the proto path.

### 7. CAST gap-fills (`ce0e2852`)

`CAST(BOOL AS STRING)` returned nil at the cascades fold path while
runtime `functions.CastValue` returned "true"/"false" — fold-vs-
runtime divergence on a constant input. Added the BOOL → STRING
arm (matches Java's CastValue.BOOLEAN_TO_STRING lowercase) and the
BOOL → FLOAT arm (mirrors the runtime's CAST(b AS INT) AS FLOAT
chain in one step).
