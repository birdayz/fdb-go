## What was done

### Bug fixes (8)

1. **Bounded aggregate index scan** — WHERE on group key pushes equality predicates into AISCAN range. `findFullScanThroughFilter` + `buildAggScanPrefix` extract Filter predicates from GroupBy's inner Reference. `WHERE status = 'pending' GROUP BY status` now uses `AISCAN [EQUALS 'pending']`.

2. **Derived table outExpr alias** — `SUM(x)/COUNT(*) AS alias` through derived tables returned NULL. Post-aggregation projection for `outExpr` columns didn't pass aliases to `NewProject`. Fixed by collecting `outName` aliases in the outExpr projection path.

3. **SUM(expr+const)** — `SUM(qty+1.0)` returned NULL. `parseAggregateText` created `FieldValue{"QTY+1.0"}` which can't resolve. Fixed: `parseOperandValue` splits arithmetic into `ArithmeticValue` with `FieldValue`/`ConstantValue` children. `OperandName` on `AggregateSpec` carries canonical text for result-map keying.

4. **SUM(col*col)** — `SUM(price*qty)` accumulated correctly but projection couldn't read the value. Spaced key variant in `finalizeGroup` bridges the gap between `aggResultName`'s space-stripped format and `ExplainValue`'s spaced format used by projection FieldValues.

5. **COUNT() empty parens** — silently became COUNT(*). Now errors at DDL parse time.

6. **mergeSortCursor.extractKey** — used pipe delimiter (collision risk). Now uses `tuple.Pack`.

### Infrastructure (2)

1. **advancePlannerStage attempt #7** — three sub-approaches tried (clear extra logicals, clear physical + re-fire BatchA, fix reoptimizeRecursive), all reverted. Root cause confirmed: inseparable from BatchA migration. Infrastructure landed: `Reference.AdvancePlannerStage(keep)`, `PlanningExplorationRules()`.

2. **Per-column selectivity heuristics** — `IndexColumnSelectivity(N, isEquality, isUnique, hasRealStats)` with 1/N, 1/sqrt(N), 0.33. NOT activated in HintCost (disrupts EXPLORE winner cascade). Blocked on BatchA migration.

### Cleanup

- Remove duplicate `concatResultCursor`, use `concatCursor[QueryResult]`

7. **HAVING SUM(expr)** — `HAVING SUM(qty*price) > 100` returned 0 rows. `rewriteAggregateValue` converted `AggregateValue` to `FieldValue` using `ExplainValue` format (`SUM((QTY * PRICE))`) with spaces and double parens. Fixed by building the name from `Op.Symbol()` + inner `ExplainValue` with spaces stripped and outer parens removed.

8. **GROUP BY with table alias returns NULL group key** — `GROUP BY o.customer` (with table alias `o` on `FROM orders o`) collapsed all rows into a single group with NULL key. The parser stored the group key as `"O.CUSTOMER"` which couldn't match the row map's bare `"CUSTOMER"` key. Fix: strip table alias prefix from GROUP BY keys in `visitSelectGroupBy`, only for single-table queries (no joins). JOINs keep qualified keys since they're meaningful for disambiguation.
