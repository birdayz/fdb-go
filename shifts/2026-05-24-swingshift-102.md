## What was done

### Bug fixes (3 — found via Java test porting)
1. **HAVING SUM(col) returned 0 rows** — aggregate index cursor named column `"AMOUNT"` instead of `"SUM(AMOUNT)"`, mismatching HAVING evaluator's lookup key
2. **WHERE on GROUP BY key returned 0 rows** — `ImplementIndexScanRule` treated aggregate index as value index, producing IndexScan + StreamingAgg (double-aggregation). Fixed by skipping `AggregateIndexMatchCandidate`
3. **COUNT(*) vs COUNT(col) wrong index selected** — when both exist, `MatchesGroupBy` matched COUNT(col) index for COUNT(*) queries because it didn't check `aggColumn==""`. Result: wrong count when NULLs present
