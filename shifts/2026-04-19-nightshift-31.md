## What was done

### SELECT DISTINCT
- `simpleTable.DISTINCT()` detection in `extractSelectParts`
- `rowKey()` length-prefixed encoding (`<tag>:<len>:<bytes>|`) for collision-free dedup
- Dedup runs before ORDER BY + LIMIT (correct SQL semantics)
- 1 FDB integration test (4 rows → 2 distinct values)

### WHERE col IN (val1, val2, ...) and NOT IN
- `evalInPredicate` dispatched from `evalExprPredicate` when `Predicate()` is `InPredicateContext`
- Column name from `ExpressionAtom()`, list values from `InList().Expressions().AllExpression()`
- Short-circuits on first match; `NOT()` negation
- 2 FDB integration tests (IN, NOT IN)

### WHERE col IS [NOT] NULL / IS TRUE / IS FALSE
- `evalIsNullPredicate` dispatched when `Predicate()` is `IsExpressionContext`
- Uses `ProtoReflect().Has()` for proto2 optional field presence — unset = SQL NULL
- Handles NULL, TRUE, FALSE test values + NOT negation
- 1 FDB integration test (IS NULL + IS NOT NULL combined)

### WHERE col IS [NOT] NULL / IS TRUE / IS FALSE
- `evalIsNullPredicate` handles `IsExpressionContext` from `Predicate()` slot
- Uses `ProtoReflect().Has()` for proto2 optional presence — unset = SQL NULL
- 1 FDB integration test (IS NULL + IS NOT NULL combined)

### WHERE LIKE / NOT LIKE
- `evalLikePredicate` + `likeMatchRunes` recursive rune-level % / _ matching
- `stripStringLiteralQuotes` handles SQL string literal unescaping
- 2 FDB integration tests (LIKE + NOT LIKE)

### WHERE BETWEEN / NOT BETWEEN
- `evalBetweenPredicate` inclusive range via `compareValues`
- 2 FDB integration tests (BETWEEN + NOT BETWEEN)

### rowKey collision fix (reviewer round 1)
- Previous `\x00`-separator encoding was vulnerable to false dedup on binary strings containing `\x00`
- Fixed to length-prefixed format: `N:0:|` for NULL, `V:<len>:<type>\x00<value>|` for values
