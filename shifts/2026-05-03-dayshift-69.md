## What to work on next

1. **#17** B5 Batch C — `FinalizeExpressionsRule` + 6 missing `ImplementXxxRule`s. Key insight from reading Java: all missing rules extend `ImplementationCascadesRule` (not `ExpressionRule`). Java's Cascades has a multi-phase planner (EXPLORATION → PLANNING). During PLANNING, `ImplementationCascadesRule` produces "final" expressions that are *disentangled* from the shared expression DAG via `FinalMemoizer`. Our Go planner is single-phase. Must either: (a) add exploration/planning phase distinction, or (b) map FinalMemoizer ops onto our existing model. Start by reading: `ImplementationCascadesRule.java` (87 LOC), `FinalMemoizer.java` (157 LOC), `ExpressionPartition.java` (104 LOC), `PlanPartition.java`, then port bottom-up. (~2 shifts)
2. **#18** Continue Phase 3 correctness tests alongside #17.
3. **#23** C4 Continuation support — match Java encoding. Gate: #22 (done).
4. **#25** ORDER BY fallback removal — falls out when queries route through Cascades.
