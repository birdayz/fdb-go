## What was done

### Major features (all in `pkg/relational/core/embedded/connection.go`)

**Subquery support**
- `WHERE col IN (SELECT ...)` / `WHERE col NOT IN (SELECT ...)` — proto and map paths
- `WHERE EXISTS (SELECT ...)` / `WHERE NOT EXISTS (SELECT ...)` — wired into
  `evalExprPredicate`, `evalPredicateOnMapExpr`, `evalHaving`, and `evalExprOnMap`
- Subquery inside `CASE WHEN` conditions (enabled by ctx+conn threading)
- UPDATE / DELETE with IN / EXISTS subquery predicates

**CTE (`WITH`)**
- Multiple CTEs per query; CTE B can reference CTE A (materialised in order)
- CTE aggregates (`GROUP BY`, `HAVING`, `COUNT/SUM/MIN/MAX/AVG`, `COUNT(DISTINCT)`)
- CTE projections with scalar expressions
- JOIN on CTE works via `scanTableToMaps` CTE shortcut
- `TestFDB_CTE*` (7 tests)

**Derived tables (FROM subquery)**
- `FROM (SELECT ...) AS alias` — materialised into temporary CTE slot
- Alias required; error otherwise

**Multi-table FROM**
- `FROM a, b WHERE a.id = b.id` (implicit cross join)
- Three-table case verified; WHERE provides the join predicate
- Self-joins via `FROM Employee AS e, Employee AS m`

**Scalar functions extended**
- New: `LTRIM`, `RTRIM`, `LEFT`, `RIGHT`, `REVERSE`, `POSITION`, `SQRT`,
  `EXP`, `LN`, `LOG` (one- and two-arg)
- `LENGTH` now rune-aware (`utf8.RuneCountInString`) — consistent with SUBSTRING
- `POWER` / `POW` now returns typed error for non-numeric args (was silently coercing to 0)

**CASE / CAST extended**
- `CASE WHEN` now works in JOIN / CTE / derived-table projections (evalSpecificFunctionOnMap)
- `CASE WHEN` also usable as a WHERE predicate directly (`WHERE CASE ... END`)
- `CAST(expr AS type)` honours parametrised types (VARCHAR(255), DECIMAL(10,2))

**ORDER BY**
- `ORDER BY` on arithmetic / function expressions in CTE and JOIN paths
  (`ORDER BY UPPER(name)`, `ORDER BY a * b`)
- Plain aggregate `ORDER BY SUM(col)` continues to work via colName path
- Explicit error when ORDER BY has an expression AND the query aggregates
  (was a silent no-op before; caught by reviewer)
- Proto / single-table scan path still requires a bare column / alias

**INSERT VALUES with expressions**
- `INSERT INTO t VALUES (1 + 2, UPPER('foo'))` works
- Multi-row insert with per-row expressions

**Other**
- SELECT without FROM (constant projection, one-row result)
- Aggregate queries with ORDER BY/LIMIT on JOIN no longer ignore those clauses
  (bug fix: spurious early return removed from `execSelectJoin`)

### Architectural refactors

**ctx + conn threading** (commit 52962c2b)
- All `evalExpr`, `evalExprAtom`, `evalScalarFunctionCall`, `evalSpecificFunction`,
  and predicate helpers (`evalIsNullPredicate`, `evalLikePredicate`,
  `evalBetweenPredicate`, `evalComparisonPredicate`) now take
  `ctx context.Context, conn *EmbeddedConnection` as first two params.
- All three `context.TODO()` placeholder sites eliminated.
- Enables subqueries inside CASE conditions and scalar-function arguments.

**Unified scalar evaluator core** (commit 2df2c61e, **-356 lines**)
- Introduces `exprEvaluator` and `predicateEvaluator` function-pointer types.
- `evalScalarFunctionCallCore` + `evalSpecificFunctionCore` hold the full dispatch.
- Four public wrappers (`evalScalarFunctionCall`, `*OnMap`, `evalSpecificFunction`,
  `*OnMap`) are ~6 lines each — adapter-construction + delegate.
- `makeProtoExprEvaluator` / `makeMapExprEvaluator` centralise closure construction.
- New scalar / CASE-capable functions only need to be added once.

**aggregateMapRows extraction** (commit c109ef3c)
- GROUP BY + aggregate machinery lifted out of `execSelectJoin` into a reusable
  `(*EmbeddedConnection).aggregateMapRows` method.
- Powers JOIN aggregates, CTE aggregates, and derived-table aggregates.
- Fixed latent bug: `execSelectJoin`'s aggregate path had a spurious
  `return nil, nil` that silently bypassed ORDER BY / LIMIT / OFFSET.

## Known issues / tech debt

- **Correlated subqueries** are still not supported. `EXISTS (SELECT ... WHERE
  outer.col = inner.col)` won't see the outer row.
- **ORDER BY expression in proto / single-table-scan path** — unsupported
  (msgs released after projection). Returns an explicit error.
- **Arithmetic wrapping an aggregate in ORDER BY** (e.g.
  `ORDER BY SUM(col) * 2`) — returns an explicit unsupported-operation error
  rather than silently ignoring. Plain aggregates like `ORDER BY SUM(col)` work.
- **No scalar subqueries in expressions** — `SELECT (SELECT COUNT(*) FROM x)
  FROM y` isn't in the grammar as an expression atom.
- **`DISTINCT`** is only handled in the proto path (line 1441 in connection.go);
  JOIN and CTE paths don't deduplicate.
- **REPEAT, STRPOS** — not in the SQL grammar's recognised function names, so
  can't parse. `REVERSE` + `POSITION` do work (and are implemented).
- **`||` string concat operator** — not in the `MathOperator` grammar slot;
  use `CONCAT(a, b)`.
