## Known issues / tech debt

- `execSelect`, `execInsert`, `execDelete`, `execUpdate` load schema on every call — no caching
- ORDER BY requires projected columns (standard SQL allows ORDER BY on non-projected cols — the full fix would carry sort-only fields separately)
- WHERE/SET only supports column-on-left, constant-on-right comparisons
- UPDATE/DELETE do full-table scan + post-filter — no index pushdown
- `evalConstant` for SET expressions doesn't support arithmetic (only literals)
- `[]byte` args in `substituteParams` encoded as hex string (documented limitation)
