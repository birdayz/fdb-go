## What Was Done

### New Package: `pkg/relational/core/keyspace`
- `RelationalKeyspace` struct with `CatalogSubspace()` and `SchemaSubspace(dbPath, schemaName)` (returns error, no panic)
- `ParseDBPath()` for URI-style `/domain/db` paths
- Unit tests for all error cases and distinct subspace guarantee

### FDB-backed StoreCatalog
- `RecordLayerStoreCatalog` using FDB record store for catalog metadata persistence
- `NewRecordLayerStoreCatalog(subspace)` and `Initialize(txn)` methods
- Full CRUD: CreateDatabase, DeleteDatabase, LoadSchema, SaveSchema, DeleteSchema, ListDatabases, ListSchemas, ListSchemasInDatabase, DoesSchemaExist, DoesDatabaseExist
- ListDatabases/ListSchemas use FDB prefix scans matching Java's behavior

### DDL ConstantActions (complete)
- `RecordLayerMetadataOperationsFactory` with optional `*keyspace.RelationalKeyspace`
- `NewRecordLayerMetadataOperationsFactory` (catalog-only) + `NewRecordLayerMetadataOperationsFactoryWithKeyspace`
- `CreateSchemaConstantAction` — creates FDB record store via `recordlayer.NewStoreBuilder()`
- `DropSchemaConstantAction` — deletes FDB record store via `recordlayer.DeleteStore()`
- `SaveSchemaTemplateConstantAction`, `DropSchemaTemplateConstantAction`
- `CreateDatabaseConstantAction`, `DropDatabaseConstantAction`

### InMemoryStoreCatalog Fixes
- Column names aligned to match FDB-backed impl: `ColDatabaseID`, `ColTemplateName`, `ColTemplateVersion`
- `ListSchemas` / `ListSchemasInDatabase` now return 4-column rows
- **Data race fixed**: `ListSchemasInDatabase` builds rows inside mutex (was reading map after unlock)

### FDB Integration Tests
- `fdb_ddl_test.go` — full end-to-end FDB tests with real keyspace isolation
- **Subspace ordering bug fixed**: catalog must use `ks.CatalogSubspace()` not root subspace (schema string-tuples land before catalog header int-tuples in byte order)
- Proper `context.WithTimeout` everywhere (no bare `context.Background()`)
