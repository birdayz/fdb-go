## What was done

### Bug fixes (4)

1. **PredicatePushDownRule existential guard** — removed overly conservative guard that blocked ALL predicate push-down when existential quantifiers were present. Java processes ForEach quantifiers independently of existential siblings. Found via porting Java's `PredicatePushDownRuleTest`. Verified NormalizePredicatesRule's guard IS needed (removing it breaks `TestFDB_CorrelatedExistsSelfJoin`).

2. **Computed expression column naming** — Java names unnamed computed expressions `_0`, `_1`, `_2` (positional). Go was using expression text like `(ID + 10)`. Fixed via dual-key approach: projection executor stores values under BOTH the ExplainValue key (internal resolution) and positional `_N` key (Java-compatible lookup). Cross-engine spec `select_constant_expr` now passes (491/492, was 490/492).

3. **PromoteValue constant folding** — `Promote(constant) → constant` with target type preserved. Matches Java's `EvaluateConstantPromotionRule`.

4. **CastValue constant folding** — `Cast(constant) → constant` evaluated at simplification time. Panic-safe via recover for out-of-range casts.

### Performance (2)

- **NLJ inner-scan caching** — uncorrelated nested-loop-join scans the inner table once instead of N times. Reduces FDB reads from N×M to 1×M for N outer rows.
- **Covering index cursor** — `coveringIndexCursor` constructs `QueryResult` from `IndexEntry.IndexValues()` without `LoadRecord()`. Infrastructure only — not yet reachable from SQL (documented in DIVERGENCES.md with precise root cause).

### Infrastructure (3)

- **ComparisonsProperty intersection semantics** — `EvaluateComparisons` returns `[]Comparison` and intersects for `IntersectionExpression` nodes. 6 tests.
- **Aggregate emission default case** — explicit error for unknown aggregate functions.
- **coveringIndexCursor interface assertion** — compile-time `RecordCursor[QueryResult]` guarantee.
