## What was done

### 1. Bug fix: Binding stress cluster file path (REGRESSION)

The testcontainers migration in swingshift-18 introduced a bug where the cluster file path passed to the Python binding tester was relative to the workspace directory, but the tester runs with `cmd.Dir=/tmp/bt-run`. The relative path resolved to the wrong location, causing error 1515 ("No cluster file found") on **all** seeds — 100% failure rate.

Root cause: `filepath.Join(seedDir, "fdb.cluster")` produces a relative path, but `exec.CommandContext` with `cmd.Dir = btRunDir` changes the working directory for the child process, so the relative path resolves wrong.

Fix: `filepath.Abs()` the cluster file path before passing to the tester.

Verified: 100/100 API seeds + 10/10 directory seeds pass after fix.

### 3. Coverage gap investigation

Investigated the coverage gaps flagged in swingshift-18 handover:

- **readpath.go (68%)**: Uncovered lines are almost entirely network error paths (connection failures, send errors, hedge fallback, wrong_shard_server retries) that require mocking to test. The `lockAware` flag path (3 lines) is the only trivially testable gap. Not worth padding.
- **conn.go (13% own tests, ~64% transitive)**: Same pattern — writeLoop coalescing, connectionMonitor dead detection, readLoop server close, TLS handshake all require network fault injection.
- **Conclusion**: These gaps are structural — the untested paths are exercised by binding stress in production, not by unit tests.
