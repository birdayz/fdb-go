## What landed (chronological)

### Semantic package seed — `pkg/relational/core/query/semantic/`

Owns identifier normalization, catalog lookup, scope resolution, function catalogue, parse-tree bridges. Neither depends on cascades.

- **Identifier** (`identifier.go`) — case-folded value-type; map-keyable by `==`; `WasQuoted` flag. `NormalizeString` exposed for call sites that don't want the wrapper.
- **QualifiedName** (`qualified_name.go`) — dot-separated; per-segment quoting; `ParseQualifiedName`, `FromSegments`, `EqualsIgnoreQuoting`, `PrefixedWith`, `LeafIdentifier`.
- **Parse bridges** (`parse_bridge.go`) — `FromFullIdContext` / `FromUidContext` read ANTLR's `GetText()` and normalize.
- **Catalog / Table / Column** (`catalog.go`) — interfaces + `InMemoryCatalog` + `StaticTable` for tests. Catalog has `LookupTable`, `TableExists`, `AllTableNames`.
- **Analyzer** (`analyzer.go`) — `ResolveTable`, `ResolveTableRef` (parse-tree form), `ResolveColumn`, `ResolveColumnRef` (bare-or-qualified dispatch), `ExpandStar`, `ExpandScopeStar`, `ExpandQualifiedStar`. Typed errors: `TableNotFoundError`, `ColumnNotFoundError`.
- **Scope** (`scope.go`) — FROM-clause tracking + correlated subquery parent chain. `ResolveColumn` with ambiguity detection, `ResolveQualifiedColumn`, `AllSourcesRecursive`. Typed errors: `AmbiguousColumnError` (carries conflicting `Sources []Identifier`), `SourceNotFoundError` (carries `Available []Identifier`), `DuplicateAliasError`.
- **Scope builder** (`scope_build.go`) — walks ANTLR `IFromClauseContext`, produces Scope. Rejects JOIN / subquery-in-FROM with `UnsupportedFromShapeError`. Handles AS-optional implicit alias (`FROM users u`).
- **Function catalogue** (`functions.go`) — `FunctionSpec` with `FunctionKind` (explicit values for serialization stability), `MinArgs`/`MaxArgs`, `AllowsStar`, `AllowsDistinct`. `RegisterDefaults` seeds COUNT/SUM/MIN/MAX/AVG. `ValidateArity` with star-case doc contract.
- **Integration test** — full parse → resolve → scope → column path.
- **Microbenchmarks** — hot-path baselines.

### Expression resolver — `pkg/relational/core/query/expr/`

New subpackage bridging semantic + cascades. Neither base package depends on the other; `expr` sits above both.

- **Resolver** bound to an Analyzer + Scope.
- **ResolveIdentifier** (bare or qualified) → cascades.FieldValue with ValueType inferred from Column type string.
- **ResolveConstant** — Go-native literals → ConstantValue / NullValue / BooleanValue. Float literals get a specific error pointing to the Phase 4.0 Type hierarchy port.
- **ResolveArithmetic** / **ResolveComparison** / **ResolveAnd** / **ResolveOr** / **ResolveNot**.
- **ResolveIn**, **ResolveCast**, **ResolveIsNull** / **ResolveIsNotNull**, **ResolveLike**, **ResolveStartsWith**.
- **ResolveFunctionCall** — `AggregateValue` dispatch via FunctionCatalog. Star handling, arity validation, typed errors.
- **WalkExpression / WalkPredicate** — the parse-tree driver! Converts an ANTLR IExpressionContext into a cascades.Value / QueryPredicate. Supports:
  - Bare column / qualified column → FieldValue.
  - Integer / string literal → ConstantValue (with escaped-quote handling).
  - Binary comparison (`a = b`) → ComparisonPredicate; operators `= <> != < <= > >= IS [NOT] DISTINCT FROM` dispatched from the ComparisonOperator context's terminal tokens.
  - LogicalExpression AND/OR → And/OrPredicate with left-deep chain flattening.
  - NotExpression → NotPredicate.
  - Parenthesised expressions (grammar-level 1-element RecordConstructor unwrapped).
  - IS [NOT] NULL via grammar-level IsExpression predicate.
  - BETWEEN via desugar to `AND(>=, <=)`.
  - Arithmetic operators (`+`, `-`, `*`, `/`) → ArithmeticValue. MOD / DIV not wired (cascades.ArithmeticOp doesn't expose them).
  - LIKE (no ESCAPE) → ComparisonLike.
  - IN-list (explicit values) → ComparisonIn; NOT IN wraps in Not.
  - Everything else (IN with subquery / parameter / column, LIKE ESCAPE, XOR, function calls, CAST) returns `UnsupportedExpressionShapeError` so callers fall back to the existing logical-builder path.
- **Integration test** — expr → cascades.Simplify pipeline. `(5 = 5) AND (id > 0)` folds to `ID > 0`; parsed `WHERE id = 1 AND 5 = 5` walks and simplifies to `ID = 1`.
- Microbenchmarks.

## Current state

**Ready-to-consume:** A parse-tree driver that wants to build a Value tree from an ANTLR ExpressionContext now has:
- `semantic.Analyzer` to resolve identifiers.
- `semantic.Scope` (+ `BuildScopeFromFromClause`) to track FROM sources.
- `expr.Resolver` to build Values for every expression shape (columns, constants, arithmetic, comparisons, AND/OR/NOT, IN, CAST, IS NULL, LIKE, STARTS_WITH, aggregate calls).
- `rlcatalog.Wrap / NewAnalyzer` to plug into a live `RecordMetaData`.

The pieces compose: integration test in `expr_test.go` shows resolver → `cascades.Simplify` pipeline folding a tautology end-to-end.

**Not yet written:** The parse-tree walker itself — the thing that takes an ANTLR `IExpressionContext` + Resolver + Scope and produces the Value/Predicate tree. That's mechanical dispatch (~300 lines); natural next-shift deliverable.
