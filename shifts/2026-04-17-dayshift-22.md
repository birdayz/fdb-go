## What was done

### IndexValidatorRegistry (feat)

Full per-index-type option change validation matching Java's `IndexValidatorRegistry` pattern. Replaced blanket option rejection with type-specific dispatch:

- **Base**: allows `replacedBy*`, `allowedForQuery`, dropping `unique` (not adding)
- **TEXT**: allows `omitPositions`/`aggressiveConflictRanges`, rejects tokenizer name change, rejects version downgrade
- **RANK**: rejects effective changes to `nLevels`/`hashFunction`/`countDuplicates` (default-aware comparison)
- **PERMUTED_MIN/MAX**: rejects `permutedSize` changes
- **VECTOR**: rejects 12 structural HNSW options, allows 6 runtime-only options
- **MULTIDIMENSIONAL**: rejects 6 structural R-tree options

11 new option constants, 23 tests.

### Former index addedVersion edge case (fix)

When a FormerIndex exists in new metadata but has no corresponding old index (index added and dropped between versions), Java validates `addedVersion > old metadata version`. Our code was missing this check ‚Äî silently accepting any addedVersion. Fixed to match Java line 480. 2 tests.

### Batch save Store field bug (fix)

`SaveRecordBatch`, `SaveRecordBatchInsertOnly`, and `InsertBatch` all created `FDBStoredRecord` without setting the `Store` field. This caused nil panic in `FunctionKeyExpression` (e.g., `get_versionstamp_incarnation`) during index updates on batch-saved records. Also fixed `oldRecord` in `SaveRecordBatch` for the same reason.

### TypedFDBRecordStore.ScanRecords type safety (fix)

Return type changed from `RecordCursor[*FDBStoredRecord[proto.Message]]` to `RecordCursor[*FDBStoredRecord[T]]` via MapCursor. Previously lost the type safety that the typed store is supposed to provide.

### DimensionsKeyExpression validation (fix)

Added to `key_expression_validate.go` switch ‚Äî was falling through to default (no validation). Now validates its inner WholeKey expression against the message descriptor.
