## What was done

### IndexValidatorRegistry (feat)

Full per-index-type option change validation matching Java's `IndexValidatorRegistry` pattern. Replaced blanket option rejection with type-specific dispatch:

- **Base**: allows `replacedBy*`, `allowedForQuery`, dropping `unique` (not adding)
- **TEXT**: allows `omitPositions`/`aggressiveConflictRanges`, rejects tokenizer name change, rejects version downgrade
- **RANK**: rejects effective changes to `nLevels`/`hashFunction`/`countDuplicates` (default-aware comparison)
- **PERMUTED_MIN/MAX**: rejects `permutedSize` changes
- **VECTOR**: rejects 12 structural HNSW options, allows 6 runtime-only options
- **MULTIDIMENSIONAL**: rejects 6 structural R-tree options

11 new option constants, 24 tests (including `allowOlderFormerIndexAddedVersion` permissive path).

### Batch save overhaul (feat + 3 fixes)

Complete batch save path now matches single-record `SaveRecord` semantics:

1. **Store field** — all 3 batch methods + oldRecord now set `Store` on `FDBStoredRecord`. Previously caused nil panic in `FunctionKeyExpression`.
2. **Record versioning** — all 3 batch methods now save record versions when `StoreRecordVersions` is enabled (one SET_VERSIONSTAMPED_VALUE mutation per record, no round trips).
3. **Old version loading** — `SaveRecordBatch` loads old record version for VERSION index cleanup on update. Previously left orphaned VERSION index entries.

`SaveRecordBatch` is now truly "semantically equivalent to calling SaveRecord N times" as its doc comment promises. Integration test verifies batch-saved records get versions.

### Former index addedVersion edge case (fix)

When a FormerIndex exists in new metadata but has no corresponding old index (index added and dropped between versions), Java validates `addedVersion > old metadata version`. Our code was missing this check. Fixed to match Java line 480. 3 tests.

### TypedFDBRecordStore.ScanRecords type safety (fix)

Return type changed from `RecordCursor[*FDBStoredRecord[proto.Message]]` to `RecordCursor[*FDBStoredRecord[T]]` via MapCursor. Integration test verifies type filtering and compile-time type safety.

### DimensionsKeyExpression validation (fix)

Added to `key_expression_validate.go` switch — was falling through to default (no validation). Now validates its inner WholeKey expression against the message descriptor.

### Tenant isolation tests (critical security)

10 new tests verifying tenant data isolation — no spill-over between tenants:

**FDB client** (4 tests): ClearRange isolation, atomic mutation isolation, GetRange isolation, conflict isolation (concurrent cross-tenant writes).

**Record layer** (6 Ginkgo specs): record visibility, independent stores, count isolation, index scan isolation, LoadRecord by PK, DeleteRecord isolation. All use same subspace path in different tenants.

Also audited tenant prefix implementation at all 3 layers (facade → client → wire protocol) — correct everywhere. Server-side prefix for reads, client-side 8-byte prefix at commit time for mutations and conflict ranges.
