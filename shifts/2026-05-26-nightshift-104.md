## What was done

### Architecture (2 commits)

1. **WithPlanningExpressionRules infrastructure** — Added `planningExpressionRules` field to Planner, `WithPlanningExpressionRules` setter, `firePlanningExprRules` helper (fires ExpressionRules during PLANNING with yieldFn→InsertFinal), and `PlanningDataAccessRules()` (safe subset of BatchA for PLANNING). This is the core mechanism for the BatchA→PLANNING migration. Not yet activated in production — requires per-column selectivity in the cost model for InJoin to win via promoteInJoinWinners.

2. **InJoin cost model fix** — `physicalInJoinWrapper.HintCost` was multiplying the inner plan's full standalone cardinality by inListLen, making InJoin always MORE expensive than Filter(FullScan). Fixed by using inListLen directly as output cardinality (one row per IN value for correlated equality probes). This makes the cost model correct for future selectivity activation.
