### 5 bug fixes

| Bug | Severity | Fix |
|---|---|---|
| **Proto FloatKind encoded as float64** | HIGH | `scalarToInterface` + `fieldStep.packInto`: FloatKind returns `float32` (FDB 0x20), not `float64` (0x21). Java's `Tuple.add(Float)` uses 0x20. |
| **ScanRecordsByType reverse+continuation** | HIGH | Continuation was always applied to low endpoint. Reverse scans need it on high endpoint. Caused duplicate records on paginated reverse scans. |
| **ScanRecordsByType ignored explicit type key** | MEDIUM | Used `RecordTypeIndex` instead of `GetRecordTypeKey()`. Records saved under explicit key were invisible to scan. |
| **5 header getters missing ensureStoreStateLoaded** | LOW | `GetUserVersion`, `GetMetaDataVersion`, `GetIncarnation`, `GetHeaderUserField`, `GetRecordStoreState` now call `ensureStoreStateLoaded()`. |
| **RANK online indexer idempotency** | LOW | `isIndexIdempotent` now checks `CountDuplicates` option for RANK indexes, matching Java's `RankIndexMaintainer.isIdempotent()`. |

All 5 bugs have regression tests that fail before the fix.
