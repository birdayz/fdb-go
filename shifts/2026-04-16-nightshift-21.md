### 10 bug fixes

| Bug | Severity | Fix |
|---|---|---|
| **Proto FloatKind encoded as float64** | HIGH | `scalarToInterface` + `fieldStep.packInto`: FloatKind returns `float32` (FDB 0x20), not `float64` (0x21). Java's `Tuple.add(Float)` uses 0x20. |
| **ScanRecordsByType reverse+continuation** | HIGH | Continuation always applied to low endpoint. Reverse scans need it on high endpoint. Caused duplicate records on paginated reverse scans. |
| **ScanRecordsByType ignored explicit type key** | MEDIUM | Used `RecordTypeIndex` instead of `GetRecordTypeKey()`. Records saved under explicit key were invisible to scan. |
| **5 header getters missing ensureStoreStateLoaded** | LOW | `GetUserVersion`, `GetMetaDataVersion`, `GetIncarnation`, `GetHeaderUserField`, `GetRecordStoreState` now call `ensureStoreStateLoaded()`. |
| **RANK online indexer idempotency** | LOW | `isIndexIdempotent` now checks `CountDuplicates` option for RANK indexes. |
| **ensureStoreStateLoaded error propagation** | MEDIUM | Error captured in `stateLoadErr`, propagated from 5 error-returning callers. |
| **Mutual indexer conflict avoidance** | MEDIUM | anyJumper pattern: when `InsertRange(requireEmpty=true)` returns false (range claimed), skip to next fragment. Includes infinite loop protection (1000 retry cap). |
| **Mutual indexer InsertRange return check** | MEDIUM | Was checking `err != nil` but `InsertRange(requireEmpty=true)` returns `(false, nil)`. Now checks boolean. |
| **Mutual indexer missing throttle** | MEDIUM | `buildIndexMutual` now wraps through `buildRangeWithRetries` for adaptive limit + retry. |
| **checkPossiblyRebuild WRITE_ONLY guard** | — | Initially added state check (b3ccb2c), then reverted to match Java (a48f159) after verifying header+clearAndMark are atomic. |

All conformance bugs have regression tests that fail before the fix.
