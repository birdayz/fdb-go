## What was done

### Deep Cascades infrastructure — 100% Java alignment

This shift ported the entire matching-to-compensation pipeline from Java, replacing every stub and proxy with real implementations.

### Value infrastructure (generic, exhaustive)

1. **RebaseValue** — rewrote from 13-type switch to generic approach: leaf values with correlation aliases remap directly, all non-leaf values recurse via `Children()`/`WithChildren()`. Covers all ~37 types automatically.
2. **EqualsWithoutChildren** — exhaustive dispatch for all 37 Value types with attribute-specific comparison. `reflect.TypeOf` fallback replaced with panic on unknown types.
3. **ValuesStructurallyEqual** — rewrote from 100-line switch + `ExplainValue` fallback to 15-line generic: `EqualsWithoutChildren` + recursive `Children()`. Matches Java's `semanticEquals` pattern.
4. **GetCorrelatedToOfValue** — expanded from QOV-only to all 7 correlation-bearing leaf types (QOV, QRV, ExistsValue, ScalarSubqueryValue, ObjectValue, UnmatchedAggregateValue, ConstantObjectValue).
5. **IsConstantValue** — explicitly lists all correlation-bearing types as non-constant.
6. **ExplainValue** — added UnmatchedAggregateValue rendering.
7. **ExplainValue equality** — replaced all 8 callers of `ExplainValue(a) == ExplainValue(b)` with `ValuesStructurallyEqual(a, b)`.
8. **ValueSemanticEquals** — new function with `ValueEquivalence` parameter for cross-scope comparison. Ports Java's `Value.semanticEquals(other, ValueEquivalence)`.
9. **ConstrainedBoolean** — added `IsTrue`, `IsFalse`, `ComposeWithOther`, `Filter` methods.
10. **IsFunctionallyDependentOn** — checks if all QOV leaves reference the same correlation.
11. **withChildren** — panic on unknown types instead of silent return-as-is.

### Predicate infrastructure

13. **GetCorrelatedToOfPredicate** — added ExistsPredicate handling.

### Compensation infrastructure (full Java algorithm)

17. **Compensation.Intersect** — full Java algorithm: recursive child intersection, GroupByMappings merge, predicate map intersection (keep common, amend), quantifier set operations, impossibility validation.
18. **Compensation.Union** — full port: merge predicate maps, multi-ForEach impossibility, recursive child union.
19. **IntersectCompensations / UnionCompensations** — monoid fold functions with proper identity/absorbing elements.
20. **UnmatchedAggregateValue** — new non-evaluable marker Value for unmatched aggregates. Implements NonEvaluable marker.

23. **CompensatedResult** — data holder for result compensation output.
24. **Compensation.Apply** — wired into data access path to inject residual filters.

## What to work on next

1. **SelectExpression.compensate** — full predicate compensation computation (~100 LOC Java). All dependencies are now ported (PullUp, childPartialMatchMap, PredicateMap, Compensation chain). This would enable real predicate-level compensation for residual filters after index scans.

2. **MaxMatchMap ValueEquivalence parameter** — cross-alias matching in PullUpValueMaybe. Currently uses structural matching only; Java passes ValueEquivalence. Needed for cross-scope comparison when query and candidate have different alias namespaces.

3. **PullUp.Visitor pattern** — MatchPullUp subclass, PullUpVisitor for building PullUp chains from expression trees. Requires expression visitor infrastructure.

4. **6 unported ImplementationCascadesRules** — all require RecordQueryFetchFromPartialRecordPlan (covering index fetch plan type, ~397 LOC Java). ~9-15 shifts total.

5. **Pareto filtering in MaximumCoverageMatches** — findContainingAccess logic that prunes dominated matches within the same MatchCandidate. Conservative without it (keeps all matches).
