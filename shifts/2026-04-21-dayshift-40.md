## What landed

### Real Java-alignment bug fixes

**Ambiguous unqualified column in JOIN errors 42702** (7a8407be).
Sentinel `ambiguousColumnMarker` poisoned at JOIN merge time; detected
in `evalExprAtomOnMap` (WHERE/ON/SELECT/HAVING expression paths),
SELECT-named-col projection, ORDER BY pre-validation, and the later
addition (a8d21096) for GROUP BY key lookup and aggregate-argument
lookup. SELECT * keeps its current dedupe-by-first-source behavior via
a qualified-alias fallback when the sentinel is encountered, so an
overlapping-column `SELECT * FROM a,b` still emits rows instead of
erroring. ErrCodeAmbiguousColumn now has call sites; pinned by 11
assertions in `ambiguous_column.yaml`.

**Wrong JOIN qualifier errors 42F01** (a08233a6).
`SELECT a.id, c.label FROM a, b` (no source `c`) used to silently
return b's `label` via the strip-table-prefix fallback. Added
`validQualifiers` set built from FROM-clause source aliases; when a
qualified reference's qualifier isn't in the set, error 42F01 (matches
Java's `UNDEFINED_TABLE` = 42F01). Also pinned the aliased-original-
name case (`SELECT a.id FROM a AS x` ‚Äî 42F01 because only `x` is
valid). Covered by `wrong_qualifier.yaml` (6 assertions). Scope:
`execSelectJoin` projection path only; WHERE/ON expressions still
silently fall back ‚Äî tracked for next shift.

**COUNT(\*) AS alias propagates through derived tables** (03bf528f).
`SELECT a FROM (SELECT COUNT(*) AS a FROM t) AS sub` returned NULL
because the COUNT(*) fast path hardcoded `"COUNT(*)"` as the column
name. Added `countStarAlias` capture at parse time and
`countStarOutName()` helper; all three fast-path emission sites plus
the countStar‚ÜíaggCols GROUP BY demotion now honor the alias. HAVING
keeps the canonical `"COUNT(*)"` lookup (SQL ¬ß7.12: HAVING binds
before SELECT-list aliases). `nested_derived_table.yaml` flipped from
pinning NULL (wrong) to pinning the aggregate value, plus a
reviewer-flagged regression guard for GROUP BY + alias + HAVING.

**Ungrouped projection column with GROUP BY errors 42803 (proto-path)**
(a56b076c). SQL ¬ß7.10 GR1: `SELECT id FROM t GROUP BY col1` must error
42803 when `id` is defined but not grouped. Pre-fix the emission loop
silently NULL-filled. Added a `groupByNames` check in the proto path
that fires after the fd-exists check (so undefined columns still
surface as 42703 first ‚Äî matches Java's error order). Scope: proto
path only; JOIN map-path still silently NULL-fills, tracked as
next-shift gap.

**Derived-table-as-left + comma-joined real table no longer drops
extras** (11f82e6a). `SELECT sq.x, b.idb FROM (SELECT ida AS x FROM a)
AS sq, b` returned `[1, NULL]` because the SubqueryTableItem branch
returned early before threading `extraCrossJoins` into the returned
selectQuery. Fix: thread them. Derived-table on the right of a
comma-join remains a separate gap (extra-source parser rejects
SubqueryTableItem with 0A000 ‚Äî still pinned).

**Arithmetic overflow** (b0afe2ad). The
`feedback_next_shift_arithmetic_overflow` memory predated nightshift-
36, which had already landed checked int64 arithmetic in applyMathOp.
Verified: no divergence on mixed int+float paths (Java's `ADD_LD` /
`MUL_LF` use `(long)l + (double)r` without overflow check, matching
our float64 fall-through). Memory removed; `overflow_mixed.yaml`
added as a regression guard.

**Duplicate ORDER BY column errors 42701** (ac62a2e9). Java's
orderby.yamsql errors 42701 on `ORDER BY b, b`. Postgres / SQL
standard accept it, but per the 100%-Java-alignment direction we now
reject. Tracks `seenOrderCols` across the ORDER BY element loop;
rejects both the name form and the positional form. Expression-form
keys (`ORDER BY b+1, b+1`) intentionally not deduped ‚Äî they go
through the expression branch and are two distinct sort keys per
Java. Pinned in `order_by_dupe_col.yaml`.

**Fractional float ‚Üí integer assignment SQLSTATE tightened to 22000**
(cd506c95). `UPDATE tbl SET bigint_col = CASE ... THEN 1 ELSE 2.2 END`
previously errored 22023 INVALID_PARAMETER; Java's case-when.yamsql
uses 22000 CANNOT_CONVERT_TYPE. Both are class-22 but the specific
code matters. `update_case_when.yaml`'s probe was fixed to actually
exercise the mixed-branch path (prior test data didn't hit the 2.2
branch); `insert_select.yaml`'s AVG-of-(10,5) case also tightened.

**UNION quantifier-aware arity error** (17be4e02). Java splits:
UNION ALL arity mismatch ‚Üí 42F64 (class-42 data error); UNION
(implicit DISTINCT) arity mismatch ‚Üí 0AF00 (FEATURE_NOT_SUPPORTED ‚Äî
set-membership is undefined with different arities). We were
returning 42F64 for both. `union_columns.yaml` pinned both codes.

**UNION incompatible-column-types errors 42F65** (eddf3103 + follow-
up). Initially pinned as a divergence, then implemented a best-effort
runtime probe: sample the first non-NULL value from each UNION side
per column and require `valuesComparable`. All-NULL columns skip the
check ‚Äî we can't infer a type without a schema-typed plan. Narrower
than Java's plan-time check but catches the common BIGINT-vs-STRING
shape. `ErrCodeUnionIncompatibleColumns` now has call sites.

**qualifier.\* + GROUP BY errors 42803** (70dc9bbc). Java's
select-a-star.yamsql errors 42803 for `SELECT a.* FROM a GROUP BY a1`
and `SELECT a.*, b.* FROM a,b WHERE a.a1=b.b1 GROUP BY a1`. Pre-fix
Go erred 0A000 with "cannot mix qualifier.* with GROUP BY". Switched
to ErrCodeGroupingError with accurate message.

### Test-coverage extensions

- **`boolean.yaml`**: added `b AND NULL` / `b OR NULL` Kleene
  truth-table rows to match Java's boolean.yamsql.
- **`cast.yaml`**: added Java cast-tests.yamsql scalar scenarios
  (CAST in projection, WHERE, aggregate argument, arithmetic over
  CAST). Array CAST remains out of scope ‚Äî Go schema has no array
  type.
- **`nested_derived_table.yaml`**: reviewer-flagged GROUP BY + alias +
  HAVING regression guard (case passes today via the existing
  `harvestAggregates` pass that dedupes COUNT(*) by its canonical
  name ‚Äî documented so a future refactor doesn't silently regress it).
- **`update_case_when.yaml`**: fixed the mixed-int-float CASE probe to
  actually exercise mixed branches. Pin documents the still-remaining
  assignment-type-narrowing gap (Java errors 22000; Go accepts).
- **`group_by_validation.yaml`** (last add): `GROUP BY col1 AS x`
  silent acceptance pinned ‚Äî Java treats AS in GROUP BY as a name-
  aliasing extension and errors 42702 on duplicate aliases. Our
  grammar silently accepts and discards. TODO in TODO.md.
