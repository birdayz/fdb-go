## What was done

### 1. Connection pool same-port aliasing fix (PR #61)

Cherry-picked from PR #61. `getOrDialConn` reused TCP connections for different IPs sharing the same port — broke multi-node FDB clusters where coordinator/GRV proxy/commit proxy all use port 4500 on different hosts. Removed same-port matching and coordinator dial fallback. Each unique ip:port now gets its own TCP connection, matching C++ FlowTransport's one-Peer-per-NetworkAddress model.

**Added regression test:** 3-container FDB cluster (172.16.1.{2,3,4}:4500) with 20 write+read transactions + range scan through the Go client. Uses `RunCluster(ctx, 3)` from the existing testcontainers module. Proves bridge IPs are routable from Bazel's linux-sandbox on Linux.

### 2. Pool read conflict buffers (-3-4 allocs/op)

`addReadConflictForKey`/`addReadConflict` used `make()` per call. Now share the same pooled `conflictBuf` as write conflicts via extracted `conflictBufAlloc` helper. Deduplicates growth logic across all 4 conflict functions (was 2 copies, now 1 helper).

| Benchmark | Before | After |
|---|---|---|
| SaveRecord | 101 allocs | 97 allocs |
| LoadRecord | 84 allocs | 81 allocs |
| DeleteRecord | 94 allocs | 91 allocs |
| ScanRecords | 1330 allocs | 1326 allocs |

### 3. AutoContinuingCursor transaction_timed_out recovery

When FDB's 5-second hard timeout (error 1031) hits mid-scan, AutoContinuingCursor now treats it as retryable by creating a new transaction from the saved continuation. Previously, 1031 escaped as non-retryable, killing large scans. This is cursor-specific handling — 1031 is NOT generally retryable, but AutoContinuingCursor creates fresh transactions. Java has the same gap.

Test with `faultAfterNCursor` wrapper: injects 1031 after 3 records in a 10-record scan, verifies all 10 returned with exactly 2 generator calls.

### 4. govulncheck in CI

Added `govulncheck ./...` step after build/test (continue-on-error: true, informational). Added `just vulncheck` recipe. Current findings: 2 vulns in `github.com/docker/docker@v28.5.2` (testcontainers transitive dep, no upstream fix available).

### 5. Binding stress testcontainers migration

Replaced raw Docker CLI calls (`docker run/exec/inspect/rm`) with testcontainers module. Eliminates manual 3-second sleep, 15-retry polling, and fragile container lifecycle. `WithDirectIP()` avoids DNAT assertion spam in the C binding used by the Python tester. Net -37 lines with better reliability.

### 6. Reviewer bug fix: write conflict buffer zero

PR reviewer found missing `buf[2*n] = 0` in `addWriteConflictForKey` — pooled buffer could have stale data, making write conflict range end key wider than intended. Safe (only causes false retries, not data corruption) but incorrect. Fixed.

### 7. C++ conformance audit (6-subsystem, 5 subagents + self)

Full audit of transaction lifecycle, RYW, GetRange multi-shard, commit path, GRV+topology, and connection/transport. 8 divergences found and fixed:

| # | Fix | Severity |
|---|---|---|
| 1 | `reset()` clear `userSetReadVersion` | Bug — stale version survives retry |
| 2 | `reset()` clear `nextWriteNoConflict` | Bug — first retry write skips conflict |
| 3 | `tryCache()` SYSTEM_IMMEDIATE guard | Maintenance hazard |
| 4 | `commitDummyTransaction` add Set mutation | C++ match |
| 5 | `commitDummyTransaction` exponential backoff | C++ match |
| 6 | `commitDummyTransaction` CAUSAL_WRITE_RISKY | C++ match |
| 7 | Location cache narrow invalidation | Performance — less cache thrash |
| 8 | Write conflict buffer zero (from PR review) | Bug — stale byte in pooled buffer |

Also fixed: deep-copy selfConflicts in commit_unknown_result path — KeyRange slices pointed into `conflictBuf` which gets reused after `reset()`, silently corrupting self-conflict data if retry writes different keys.

**Divergence table: 16/17 fixed, 1 remaining (auto-reset DESIGN choice).**

Confirmed correct (zero divergences): RYW (all 12 atomics), GetRange multi-shard (more flag, reverse, selectors), commit wire format, GRV per-priority cache, topology updates, connection lifecycle.
