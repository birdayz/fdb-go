## What was done

### Bug fixes (4)

1. **DeleteRecordsWhere leaked index clears to non-target record types** — `findMatchingRecordTypes()` only checked PK column count, not the record type key value. When deleting Orders via type key prefix, Customer-only indexes were incorrectly cleared. Fix: filter by type key VALUE when PKs have RecordTypeKey prefix, with Go int type normalization. Test: "skips type-specific index that doesn't cover deleted type".

1b. **computeIndexDeletePrefix used arbitrary sample PK** — picked first record type from `md.RecordTypes()` (non-deterministic map iteration). Now uses first matching type from `matchingTypeNames`. Addresses reviewer observation.

2. **nil record panic in SaveRecordBatchInsertOnly and InsertBatch** — both accessed `records[0].ProtoReflect()` before checking for nil records. Added nil guard before compiled PK evaluator setup.

3. **rtreeScanCursor.IsClosed() always returned false** — `Close()` didn't set any state. Added `closed bool` field. Also fixed `prefixSkipScanCursor` to track own state.

### KeySpace Phase 1 (new package)

`pkg/recordlayer/keyspace/` — pure data structure implementation, no FDB dependency:
- **KeyType** enum (NULL, BYTES, STRING, LONG, FLOAT, DOUBLE, BOOLEAN, UUID) with validation
- **Directory** tree (name, type, constant value, children)
- **KeySpace** root + **Path** navigation with type checking and constant enforcement
- **Forward resolution**: Path → Tuple → Subspace
- **Reverse resolution**: PathFromTuple (Tuple → Path with remainder)
- **Range queries**: Path.ToRange() for directory subtree scanning
- **Path utilities**: Depth, ListSubdirectories, HasSubdirectory, String
- **11 unit tests** covering all features
- **Design document**: `docs/design-keyspace.md` (4-phase plan, ~7-10 shifts total)
