## Theme

Three MEDIUM-priority follow-ups from nightshift-48 plus one LOW
opportunistic gap-fill:

1. **Parameter-bound Comparison** — `?` and `?name` walk through to a
   real `cascades.ParameterValue`; constant-fold declines; ExplainValue
   renders `?N` / `?name` as the plan-cache key seam. Per-statement
   ordinal counter on Resolver assigns 1-based ordinals matching Go's
   `database/sql` `NamedValue.Ordinal`.

2. **Scalar function calls in walker** — UPPER, LOWER, LENGTH /
   CHAR_LENGTH / CHARACTER_LENGTH, OCTET_LENGTH dispatch via
   `walkScalarFunction` → `cascades.ScalarFunctionValue`. Composes
   through IsConstantValue so `UPPER('foo')` folds and
   `UPPER(field)` doesn't. Plan-time `evalScalarFunction` mirrors the
   embedded executor's semantics (utf8 rune count for LENGTH family,
   byte length for OCTET_LENGTH).

3. **Standalone Value simplification** — `SimplifyValue` is the
   free-function counterpart to `Simplify` for Values not inside a
   QueryPredicate. Two-phase post-order: simplify children, then
   collapse to a literal Value via LiteralValue if the rebuilt node
   is constant per IsConstantValue. Whitelist of foldable composites
   (Arithmetic / Cast / Promote / ScalarFunction).

4. **Integer DIV operator** — `DIV` keyword shares OpDiv with `/` at
   the seed because Go's `/` on int64 already truncates toward zero.
   Documented the float-arithmetic split point inline; pinned
   truncation parity (incl. negative dividend) and divide-by-zero NULL
   in tests.

## What landed (chronological)

### 1. cascades: `ParameterValue` + walker dispatch

* New `ParameterValue{Ordinal, ParamName, Typ}` — leaf Value,
  Children=∅, Type=TypeUnknown. Optional `ParameterBinder` eval-
  context capability resolves bindings at runtime; degrades to NULL
  otherwise.
* `IsConstantValue` rejects so constant-fold rules decline on
  `x = ?` / `x = ?name`.
* `ExplainValue` renders `?` / `?N` / `?name` (matches the lexer's
  `[?$][A-Za-z][A-Za-z0-9_/]*` NAMED_PARAMETER rule). Both `?foo`
  and `$foo` deliberately collapse to one canonical form for
  plan-cache keying.
* Walker dispatches `PreparedStatementParameterAtomContext` →
  `walkPreparedParameter`; reads QUESTION / NAMED_PARAMETER off
  the parse node. The Resolver carries `nextOrdinal int` so multiple
  `?` in one statement get distinct 1-based ordinals.
* No change to `ResolveComparison`: nightshift-48's Operand→Value
  generalisation already accepts non-constant RHS, so the seam was
  already in place.

### 2. cascades: `ScalarFunctionValue` + walker dispatch

* New `ScalarFunctionValue{FuncName, Args, Typ}` — composite Value;
  Children() returns Args. Constant-fold recurses correctly.
* `evalScalarFunction` is the inline switch — UPPER/LOWER use
  `strings.ToUpper`/`ToLower`; LENGTH family uses
  `unicode/utf8.RuneCountInString` / `RuneCount`; OCTET_LENGTH uses
  byte length. Mirrors `embedded.scalar_functions.go` semantics so
  plan-time fold and runtime eval stay consistent.
* `ExplainValue` renders `FN(args, ...)`.
* Walker dispatches `ScalarFunctionCallContext` →
  `walkScalarFunction`; function names normalised to upper-case;
  unknown names decline so the logical-builder text fallback still
  catches them.
* `scalarFunctionResultType` is the inline name → result-type catalog;
  full registry-driven dispatch lands with the function-catalog port.
* Pre-existing `TestBuildLogicalPlanWithCatalog_UnsupportedShape`
  used `UPPER(price)` as its "walker declines" probe — that shape is
  now walked. Replaced with `FROBNICATE()` (a deliberate non-existent
  function) and added
  `TestBuildLogicalPlanWithCatalog_ScalarFunctionWalked` to pin the
  new path.

### 3. cascades: `SimplifyValue` standalone constant-fold

* Free function (not a `CascadesRule` — the rule framework targets
  QueryPredicate matchers). Phase 4.6 brings
  `ValueSimplificationRuleSet` and SimplifyValue retires.
* Two-phase post-order: rebuild children via `simplifyChildren`, then
  collapse to a literal via `EvaluateConstant` + `LiteralValue` if
  `isFoldableComposite(rebuilt)` returns true.
* Whitelist of foldable composites: Arithmetic / Cast / Promote /
  ScalarFunction. Leaves and unknown composites pass through
  unchanged so callers' pointer-equality short-circuit stays useful.
* Type preservation: `LiteralValue` defaults to TypeUnknown for
  non-bool / non-nil literals; SimplifyValue patches the result to
  `v.Type()` so downstream type checks keep working. Same patching
  for NullValue (NULL fold inherits source type — round-2 follow-up).

### 4b. expr: race-detector hardening on Resolver.functionCatalog

Wind-down race-detector run (`bazelisk test --@rules_go//go/config:race`)
on cascades + expr + embedded surfaced a pre-existing data race on
`Resolver.funcCat` lazy init: `TestWalkExpression_AggregateFunctions`
runs subtests with `t.Parallel()` against a shared Resolver, and the
unsynchronised nil-check + assign in `functionCatalog` races with
`FunctionCatalog.Lookup`'s map read. Latent — only newly visible
because the targeted race invocation exercised the function-call path.

Fix: `sync.Once` wraps the lazy build so concurrent first-use is safe.
Plus an explicit concurrency-contract doc on Resolver: single-statement
/ single-goroutine (the new `nextOrdinal` counter would also race
under sharing — documented so future readers don't add more shared-
Resolver patterns).

### 4. expr/walker: `DIV` keyword → `OpDiv`

The grammar accepts both `/` and `DIV` (`RelationalParser.g4:1277`);
the walker only mapped `DIVIDE()`. Added `DIV()` to the same arm.
At the seed both forms produce OpDiv because
`ArithmeticValue.Evaluate` is int64-only and Go's `/` on int64
already truncates toward zero. Documented the float-arithmetic
split point.

## Current state

**Parameter-bound comparisons compose end-to-end.** `WHERE x = ?`
walks through to a `ComparisonPredicate` with RHS
`*cascades.ParameterValue{Ordinal: 1}`; multiple `?` in one
statement get distinct ordinals (`(ID = ?1 AND NAME = ?2)`).
Constant-fold rule declines via IsConstantValue. ExplainValue
renders the canonical `?N` form so plan-cache keying is the right
shape — different bind values at the same shape share one key.

**Scalar functions UPPER / LOWER / LENGTH / CHAR_LENGTH /
CHARACTER_LENGTH / OCTET_LENGTH walk to ScalarFunctionValue.**
Catalog-aware logical builder attaches a real Predicate carrying
the scalar-fn tree (`TestBuildLogicalPlanWithCatalog_ScalarFunctionWalked`).
`UPPER('foo')` folds at plan time via `EvaluateConstant`;
`UPPER(field)` doesn't.

**Standalone Value constant-fold via SimplifyValue.** Folds
`(1+2)*3` → `9` (full); `name + (1+2)` → `name + 3` (partial);
`CAST(1+2 AS STRING)` → `"3"`; `UPPER(LOWER('Hi'))` → `"HI"`;
`PROMOTE(1+2, TypeFloat)` → `3 :: TypeFloat`. NULL through
arithmetic propagates as NullValue with source Type carried.

**DIV keyword wired** — `id / 7` and `id DIV 7` both produce
OpDiv with truncated-toward-zero semantics.
